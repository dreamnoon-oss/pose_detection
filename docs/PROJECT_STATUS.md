# 端头门司机行为分析 — 项目进度

## 项目概述

基于 YOLOv8 姿态估计，对列车司机在站台上的标准动作进行实时识别与合规判断。当前使用上体场2视频。

## 动作规范（来自申通技术中心文档）

| 序号 | 动作名称 | 规范描述 | 关键判断条件 |
|------|---------|---------|-------------|
| 1 | 开门后手指呼唤 | 到站开门后，司机立于站台，手臂举起与肩平行，指向站内 | 维持2秒以上 |
| 2 | 手动关门 | 直线站台目视关门；弯道/CCTV辅助关门 | 手臂抬起60°左右，2秒以上 |
| 3 | 关门后确认夹缝 | 客门关闭后，手臂举起与肩平行，指向列车与站台夹缝 | 维持2秒以上 |
| 4 | 开车前确认站台指示灯 | 客门关闭后，手臂举起≥肩部，指向站台指示灯 | 维持1秒以上 |
| 5 | 开车前确认道岔（可选） | 手臂举起与肩平行，指向前方轨道岔路口 | 维持1秒以上 |

不规范：司机动作过快、动作变多或变少。

### 模块职责

| 模块 | 职责 |
|------|------|
| `config.py` | 集中管理路径、关键点定义、骨架连接、颜色、阈值常量 |
| `geometry.py` | `angle_between()` / `segments_intersect()` / `min_angle_to_rect()` |
| `detection.py` | 3种检测策略：`check_arm_parallel_to_line()` / `check_arm_passes_region()` / `check_pointing()` + `check_pointing_with_line()` |
| `state_machine.py` | `ActionStateMachine` 类：可配置动作序列，支持 4 种检测类型 |
| `visualization.py` | 骨架绘制、中文 PIL 文本渲染、状态面板、帧信息叠加 |
| `annotation.py` | JSON 加载/保存、ROI 框选、参考线绘制 UI |
| `player.py` | `VideoPlayer` 类：视频 I/O、模型推理、交互控制、渲染管线 |

### 检测策略

| 类型 | 算法 | 使用场景 |
|------|------|---------|
| `parallel_line` | 手臂方向(肩→腕)与参考线夹角 < 阈值 | 动作1/2/3（手指呼唤、关门、确认夹缝） |
| `pass_region` | 肩→腕线段与矩形区域相交或被包含 | 动作4（确认指示灯） |
| `pointing` | 手臂方向与区域4角+中心的最小夹角 < 阈值 | 备选方案 |
| `pointing_with_line` | 手臂平行于线 + 大致朝向区域 | 双重判断备选 |

### 动作状态机（上体场2 — 当前4动作）

| 动作 | 判断逻辑 | 目标对象 | 备选策略 |
|------|---------|---------|---------|
| 动作1 | `parallel_line`（肩→腕与线平行） | line_1 | 无 |
| 动作2 | `parallel_line`（肩→腕与线平行） | line_2 | 手腕检测不到时回退用肘部 |
| 动作3 | `parallel_line`（肩→腕与线平行） | line_1 | 无 |
| 动作4 | `pass_region`（肩→腕穿过区域） | region_1 | 无 |

### 动作状态机（宝山1 — 5动作，角度法）

| 动作 | 判断逻辑 | 目标对象 |
|------|---------|---------|
| 动作1 | `pointing_with_line` | region_1 + line_1 |
| 动作2 | `pointing` | region_2 |
| 动作3 | `pointing_with_line` | region_1 + line_1 |
| 动作4 | `pointing` | region_3 |
| 动作5 | `pointing_with_line` | region_4 + line_1 |

## 技术参数

- 模型：`yolo26x-pose.pt`
- 关键点：5-12（肩/肘/腕/髋）
- 平行角度阈值：40°
- 持续帧数：15帧
- 帧衰减：每帧 -2（容忍短暂丢帧）
- 最小手臂长度：30px
- 置信度阈值：0.5

## 标注工具操作

| 按键 | 功能 |
|------|------|
